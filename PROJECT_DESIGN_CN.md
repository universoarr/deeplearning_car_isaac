# Isaac Lab 小车智能行驶项目设计

## 1. 项目目标

在 Isaac Lab 中训练一台带有 IMU 六轴传感器和前视摄像头的小车，使其在复杂地形中具备以下能力：

- 在崎岖、碎石、轻微坡地等非平整路面上保持稳定直线行驶
- 识别前方障碍并自主绕行
- 在存在扰动、打滑、颠簸的情况下维持较好的姿态与方向控制
- 具备一定泛化能力，从训练地形迁移到未见过的复杂场景

## 2. 建议拆成两个任务

不要一开始就直接做“端到端复杂场景智能驾驶”，建议拆成两个可训练任务：

### 任务 A：地形稳态直行

目标：

- 给定期望航向，车辆在颠簸地形上尽量保持直线
- 抑制横摆、侧滑和大幅姿态抖动

输入建议：

- IMU：角速度、线加速度
- 车体状态：速度、角速度、姿态
- 历史动作：最近 2 到 4 帧油门/转向

输出建议：

- 连续动作：`[throttle, steering]`

### 任务 B：感知驱动避障导航

目标：

- 在前方有石块、路障、坑洼边缘时绕行
- 绕开障碍后继续回到主方向

输入建议：

- 前视相机图像
- IMU
- 当前速度和航向误差

输出建议：

- 连续动作：`[throttle, steering]`

## 3. 为什么这样拆

如果直接把相机、IMU、复杂地形、避障、路径跟踪全部揉进一个任务，训练会很容易出现：

- 奖励稀疏
- 收敛慢
- 策略先学会“停车保平安”
- 视觉特征还没学稳，控制器已经发散

更稳妥的路线是：

1. 先用低维状态把底盘控制练稳
2. 再把视觉加入到避障任务
3. 最后再做融合任务或分层控制

## 4. 推荐算法路线

## 第一阶段：低维状态强化学习

用于“崎岖地面保持直线”：

- PPO：最稳妥，适合作为第一版基线
- SAC：连续控制效果好，但工程复杂度略高

建议优先：

- `PPO + MLP policy`

观测：

- 车体线速度、角速度
- 姿态或重力投影
- IMU 原始或滤波后的六轴数据
- 目标航向误差
- 上一时刻动作

## 第二阶段：视觉避障

用于“看见障碍并绕行”：

- PPO + CNN encoder
- 或者先做 imitation learning / behavior cloning 预训练，再接 PPO 微调

建议优先：

- `CNN + MLP + PPO`

如果训练不稳定，可改成：

- 用教师策略或传统规划器生成示范
- 先监督学习，再强化学习 fine-tune

## 第三阶段：多模态融合

推荐结构：

- 图像分支：CNN / ResNet18-lite
- IMU 与状态分支：MLP
- 融合层：concat 后接 policy/value head

## 6. 传感器设计建议

### IMU

保留以下观测：

- `angular_velocity`
- `linear_acceleration`
- 可选：重力在车体坐标系下投影

用途：

- 判断车体是否发生横摆、俯仰、侧倾
- 在视觉信息不足时维持稳定控制

### Camera

推荐先用单目灰度或 RGB 前视相机：

- 分辨率先从 `84x84` 或 `128x128` 开始
- 视场角不要太窄
- 摄像头相对车头略微俯视

用途：

- 感知前方障碍
- 判断通行区域
- 学习绕行动作

## 7. 奖励函数设计

### 任务 A：崎岖地形直行

正奖励：

- 前向速度接近期望值
- 航向误差小
- 横向偏移小

惩罚：

- 过大转向
- 频繁动作抖动
- 横摆角速度过大
- 翻车或车体离地异常
- 长时间停滞

可以写成加权和：

```text
reward =
  + w1 * forward_velocity
  - w2 * heading_error
  - w3 * lateral_offset
  - w4 * yaw_rate
  - w5 * action_smoothness
  - w6 * roll_pitch_penalty
```

### 任务 B：避障

正奖励：

- 朝目标方向推进
- 成功绕开障碍
- 恢复到主路径

惩罚：

- 撞击障碍
- 距障碍过近
- 原地打转
- 无效绕远

## 8. 终止条件

- 车辆翻倒
- 碰撞强度超过阈值
- 偏离任务区域
- 长时间没有前进
- 超过最大 episode 长度

## 9. 训练路线图

### Phase 1：底盘控制基线

目标：

- 不用相机，只靠状态和 IMU，在随机起伏地面直行

验收标准：

- 多种地形下保持稳定前进
- 航向偏差和翻车率明显下降

### Phase 2：加入障碍但先不用端到端视觉

目标：

- 可先用简化障碍距离观测做避障

验收标准：

- 策略学会绕开单个、多个障碍

### Phase 3：相机端到端避障

目标：

- 用前视图像替代简化距离观测

验收标准：

- 对随机障碍和不同光照具有鲁棒性

### Phase 4：多任务融合

目标：

- 在崎岖路面上同时保持稳定与避障

验收标准：

- 未见地形上仍能完成较长距离自主通行

## 10. 最推荐的技术方案

如果你现在想尽快做出第一版可运行成果，我建议：

- 仿真平台：Isaac Lab
- 控制对象：Ackermann 或差速小车
- 观测：
  - 第一版只用 IMU + 车体状态
  - 第二版再加前视相机
- 算法：
  - 第一版 `PPO`
  - 第二版 `CNN + PPO`
- 训练策略：
  - 先单任务
  - 再课程学习
  - 再域随机化

4. 一个最小可训练的 Isaac Lab RL env

## 12. 一个务实的最小可行版本 MVP

第一版先只做：

- 地形：轻微起伏 + 碎石障碍
- 传感器：IMU
- 目标：保持直线前进 5 到 10 米
- 算法：PPO

第二版再加：

- 前视相机
- 障碍绕行

这样成功率最高，也最容易调试。
